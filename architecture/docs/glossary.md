# Glossary (Глоссарий проекта)

## AI Core
Центральная логическая часть системы, содержащая правила, модели, ограничения и архитектурные принципы.

## Boundary
Логическая или техническая граница, ограничивающая доступ ИИ к ресурсам, данным или действиям.

## Trust Boundary
Граница доверия, регулирующая, что ИИ может и не может делать без подтверждения человека.

## Orchestration Layer
Уровень управления процессами, автоматизацией и последовательностью действий (например, n8n).

## Interface Layer
Уровень взаимодействия с пользователем — приложение, ассистент, чат, визуальный интерфейс.

## Tools Layer
Слой инструментов, утилит, интеграций и подключаемых сервисов.

## System Context
Описание окружения системы, внешних акторов и взаимодействий.

## Use Case
Сценарий использования системы в реальной задаче.

## Requirements
Функциональные требования — что система должна уметь.

## Non-Functional Requirements
Нефункциональные требования — скорость, надёжность, безопасность, UX.

## Architecture Guidelines
Набор правил и ограничений, по которым строится система.

## Security Model
Модель обеспечения безопасности, описывающая угрозы, защиту, ответственность и контроль.

## Human Override
Принцип, по которому человек всегда имеет приоритет и право остановки.

## Safe Defaults
Политика безопасных значений по умолчанию.

## Memory Layer (External Memory)
Внешний слой памяти — хранение данных вне ИИ под контролем человека.

## Autonomy Level
Уровень автономности ИИ: от полного ручного управления до полуавтоматического режима.

## Audit Log
Невозможность скрыть или изменить действие — прозрачный журнал операций.

## Orchestration Pipeline
Цепочка задач, запускаемых в строгом порядке через n8n или другой движок.
