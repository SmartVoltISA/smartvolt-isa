# System Context

## 1. Внешние участники системы
- Пользователь — взаимодействует через интерфейсы (чат, голос, панель управления).
- Администратор — управляет доступами, политиками, логикой.
- Внешние сервисы — LLM-ядра, базы данных, API, облачные вычисления.
- Аппаратные устройства — датчики, исполнительные механизмы, интерфейсы управления.

## 2. Основные границы
Система разделена на четыре уровня:

1. Core Layer — ядро обработки, модели, интерпретация данных.
2. Orchestration Layer — управление потоками, правила, сценарии, выполнение задач.
3. Interface Layer — каналы ввода/вывода, UI, API.
4. Tools Layer — инструменты, плагины, вспомогательные модули.

## 3. Взаимодействие уровней
- Core принимает запросы, формирует смысл и решения.
- Orchestration управляет ходом сценариев и безопасностью.
- Interface обеспечивает доступ и передачу данных.
- Tools расширяют систему и дают дополнительные возможности.

## 4. Основные данные, проходящие через систему
- Команды пользователя.
- Логи и события.
- Контекст задач.
- Метаданные безопасности.
- Результаты вычислений и ответов.

## 5. Ограничения и требования
- Все операции логируются.
- Доступ — только через контролируемые интерфейсы.
- Полная прозрачность работы.
- Возможность автономного режима при сбоях.
- Внешняя память — контролируемая, безопасная, с проверкой.
