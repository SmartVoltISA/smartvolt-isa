# Trust Boundaries (Границы доверия)

## 1. Human-Control Boundary
- Человек всегда является финальным оператором.
- ИИ не выполняет критические действия без подтверждения.
- Человек может отключить ИИ в любой момент (пульт / кнопка / команда).
- Все автоматические процессы имеют ручной override.

---

## 2. Data Access Boundary
- ИИ не имеет прямого доступа к файлам пользователя.
- Вся работа идёт через контролируемые API.
- Логи доступны человеку, но ИИ видит только ограниченные данные.
- Внешние сервисы получают только минимально необходимую информацию.

---

## 3. Action Boundary
- ИИ может выполнять только разрешённые сценарии.
- Никаких «инициативных» действий.
- Любое расширение функционала требует утверждения владельца системы.
- Все операции проходят валидацию на безопасность.

---

## 4. Autonomy Boundary
- ИИ не имеет права изменять правила, инструкции, процессы.
- ИИ работает строго внутри утверждённых политик.
- Никаких обходных решений, саморедактирования и самообучения без контроля.

---

## 5. Network Boundary
- Все сетевые взаимодействия проходят через шлюз.
- Нет прямых запросов от ИИ к внешним ресурсам без фильтра.
- Внешние LLM — только через безопасный канал.

---

## 6. Hardware Boundary
- ИИ не управляет устройствами напрямую — только через контроллеры.
- Контроллеры имеют жёсткие лимиты, которые нельзя превысить.
- Критическое оборудование недоступно для автоматической работы.

---

## 7. Memory Boundary
- Память внешняя, контролируемая человеком.
- ИИ не может сохранять данные самостоятельно.
- Любое долговременное хранение проходит через модуль безопасности.

---

## 8. Audit Boundary
- Все действия ИИ логируются.
- Изменение логов невозможно со стороны ИИ.
- Человек имеет полный доступ к просмотру и проверке всех событий.
