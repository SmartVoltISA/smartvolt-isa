# Interface Layer (User Interaction Layer)

## 1. Purpose
Слой интерфейсов обеспечивает точку взаимодействия между пользователем и архитектурой ISA.  
Задача — сделать доступ к логике безопасным, контролируемым и понятным.

## 2. Components
- **Assistants** — текстовые и голосовые интерфейсы (ChatGPT, локальные LLM, голосовые агенты).  
- **Robots & Devices** — физические устройства, роботы-компаньоны, панели управления.  
- **APIs & UI Panels** — веб-интерфейсы, мобильные приложения, панели мониторинга.

## 3. Requirements
- Упрощённое взаимодействие без доступа к ядру.
- Все команды проходят через слой безопасности.
- Контроль контекста и ограничение прав.
- Логи всех действий сохраняются.

## 4. Key Functions
- Отправка запросов в оркестрационный слой.
- Получение ответов и структурирование их для пользователя.
- Безопасная маршрутизация команд.
- Управление диалоговыми контекстами.

## 5. Design Philosophy
- Интерфейс не должен «думать сам».
- Он выполняет роль проводника.
- Минимум логики внутри, максимум — в оркестрации.
